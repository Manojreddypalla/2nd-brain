Here's the **exact actionable process** Mayank describes in the video, stripped down into steps.

# 1. DSA Preparation

## Step 1: Pick one topic

Examples:

- Dynamic Programming
    
- Trees
    
- Graphs
    
- Binary Search
    
- Sliding Window
    

Study **one topic at a time**.

---

## Step 2: Solve important Medium questions first

Don't spend much time on Easy problems.

Focus on:

- Most asked LeetCode Mediums
    
- Popular interview questions
    

---

## Step 3: If stuck, look at solution

Don't endlessly struggle.

Try first.

If still stuck:

- Read solution
    
- Understand approach
    
- Understand pattern
    

---

## Step 4: Create your own pattern notes

Example:

Instead of writing entire code:

```text
DP Pattern:
- State
- Transition
- Base Case
- Memoization

Matrix DP:
- Top -> Down
- Left -> Right

Array DP:
- dp[i] depends on previous states
```

Write only the pattern.

Not the full solution.

---

## Step 5: Mark difficult questions

Create a category:

```text
TO REVISIT
```

or

```text
REPEAT LATER
```

Questions:

- Couldn't solve
    
- Solved after seeing solution
    
- Found tricky
    

---

## Step 6: Don't revisit immediately

Do NOT retry next day.

Wait:

- 2 weeks
    
- 3 weeks
    

Reason:

Otherwise you're testing memory, not understanding.

---

## Step 7: Retry without seeing solution

After 2–3 weeks:

Open question.

Try solving from scratch.

---

## Step 8: If stuck again

Don't reread everything.

Identify:

```text
Where exactly did I fail?
```

Example:

```text
I remembered DFS.

Forgot backtracking step.
```

or

```text
I remembered DP state.

Forgot transition.
```

Analyze the exact failure point.

---

## Step 9: Find similar questions

Use:

- LeetCode
    
- ChatGPT
    

Ask:

```text
Give me questions with similar pattern.
```

Practice those.

---

## Step 10: Revisit again after 1 month

Check:

```text
Did I actually learn it?
```

or

```text
Did I memorize it?
```

---

# 2. Daily DSA Revision

Every morning:

Spend:

```text
10–15 minutes
```

Review pattern notebook.

Example:

```text
DP
- State
- Transition
- Base

Trees
- DFS
- BFS

Graph
- Visited
- Queue
```

Quick scan only.

No heavy studying.

---

# 3. System Design Preparation

## Step 1: Learn connected architecture

Don't study:

```text
Redis
Kafka
Load Balancer
Database
```

as isolated topics.

Understand:

```text
How everything connects together.
```

Start from:

```text
Client → Server
```

Then scale.

---

## Step 2: Make notes

For every architecture:

Example:

```text
Uber
Google Docs
WhatsApp
Netflix
```

Keep notes.

---

## Step 3: Focus on core component

Don't waste revision on common pieces.

Everybody already knows:

```text
Redis
Load Balancer
Database
CDN
```

Focus on:

### Uber

```text
How rider maps to driver
```

### Google Docs

```text
Conflict resolution
```

### WhatsApp

```text
Message delivery
```

### YouTube

```text
Video serving
```

Focus on unique logic.

---

## Step 4: Save diagram

Keep:

```text
Architecture diagram
```

Review occasionally.

# 7. Main Rule

The entire video boils down to:

```text
Learn
↓
Identify Pattern
↓
Write Pattern
↓
Revise Pattern
↓
Retry Question After 2–3 Weeks
↓
Analyze Failure Point
↓
Practice Similar Questions
↓
Revise Again
```

His central message is:

> Don't try to remember solutions. Remember patterns. And don't trust that you've learned something until you can solve it again weeks later without looking at the answer.
