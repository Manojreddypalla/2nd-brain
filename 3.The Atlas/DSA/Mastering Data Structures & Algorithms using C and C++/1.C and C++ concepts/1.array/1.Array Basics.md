# 2. Core Idea of Arrays

## Definition

```
Array:
A collection of similar data elements stored together using one name.
```

Example:

```c
int A[5];
```

This creates:

- 5 integer locations
- one common name → `A`
- indices from `0` to `4`

---

# 3. Why Arrays Exist

Without arrays:

```c
int a1,a2,a3,a4,a5;
```

This becomes hard to manage.

Arrays solve this by:

- grouping data
- easy traversal
- contiguous memory storage
- fast indexing

---

# 4. Important Properties of Arrays

## Similar Data Type

All elements must be same type.

Example:

```c
int A[5];
```

Only integers allowed.

---

## Continuous Memory

Arrays are stored continuously in memory.

Example:

```
A[0] A[1] A[2] A[3] A[4]
```

This makes access very fast.

---

## Index Starts From 0

For size 5:

```
0 1 2 3 4
```

Last index:

```
size - 1
```

---

# 5. Declaring Arrays

## Syntax

```c
datatype arrayName[size];
```

Example:

```c
int A[5];
```

Creates:

- 5 integer spaces

---

# 6. Memory Size of Array

Formula:

```
Total Size = number of elements × size of datatype
```

Example:

```c
int A[5];
```

If integer = 4 bytes:

```
5 × 4 = 20 bytes
```

The lecture demonstrated using:

```c
sizeof(A)
```

Output:

```
20
```

---

# 7. Accessing Array Elements

Using index:

```c
A[0] = 12;
A[1] = 15;
A[2] = 25;
```

Meaning:

- store values at specific locations

---

# 8. Array Initialization

## Method 1 — Manual Assignment

```c
A[0]=2;
A[1]=4;
```

---

## Method 2 — Direct Initialization

```c
int A[5]={2,4,6,8,10};
```

This:

- creates array
- fills values immediately

---

# 9. Size Can Be Omitted

```c
int A[]={2,4,6,8,10};
```

Compiler automatically calculates size.

Size = number of elements.

---

# 10. Partial Initialization

Example:

```c
int A[10]={2,4,6};
```

Remaining elements become:

```
0
```

Final array:

```
2 4 6 0 0 0 0 0 0 0
```

Very important concept.

---

# 11. Initializing Entire Array With Zero

```c
int A[10]={0};
```

All elements become zero.

---

# 12. Garbage Values

If array is NOT initialized:

```c
int A[5];
```

Then memory contains:

```
garbage values
```

Meaning:

- random old memory data
- not assigned by programmer

Important:

```
Garbage can also be 0 sometimes.
```

But it is still garbage because YOU did not assign it.

This is a very important beginner concept.

# 14. Using Loops With Arrays

## Why?

Arrays usually contain many elements.

Loops help access all elements automatically.

---

# 15. For Loop Traversal

```c
for(int i=0;i<5;i++)
{
    printf("%d",A[i]);
}
```

Process:

- i = 0 → prints A[0]
- i = 1 → prints A[1]
- continues...

---

# 16. For Each Loop (C++)

C++ supports:

```c
for(int x:A)
{
    cout<<x;
}
```

Meaning:

- `x` takes each element one by one

This is cleaner for traversal.

---

# 17. Variable Size Arrays

Example:

```c
int n;
cin>>n;
int A[n];
```

Size depends on user input.

---

# 18. Problem With Variable Size Arrays

They usually:

- cannot be initialized directly
- may behave differently in compilers

Example not preferred:

```c
int A[n]={2,4,6};
```

Can give:

- compiler error
- improper behavior

Instructor avoids this style mostly.

# 20. Important Observations

## Arrays Are Fast

Because:

- direct indexing
- contiguous memory

---

## Fixed Size Problem

Normal arrays:

- size fixed during creation

Cannot dynamically grow easily.

This becomes important later in:

- dynamic arrays
- linked lists

---

# 21. Common Mistakes

## Mistake 1

```c
for(i=0;i<=5;i++)
```

Wrong for size 5 array.

Correct:

```c
for(i=0;i<5;i++)
```

---

## Mistake 2

Using uninitialized arrays.

Leads to garbage values.

---

## Mistake 3

Accessing invalid index:

```c
A[10]
```

for size 5 array.

This causes undefined behavior.
