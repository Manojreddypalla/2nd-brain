# Backtracking — The Actual Idea

Most students memorize:

```text
Choose
Explore
Backtrack
```

But that doesn't build intuition.

Let's build the real mental model.

---

# Imagine You're In A Maze

You enter a maze.

At every intersection:

```text
Go Left?
Go Right?
Go Straight?
```

You don't know which path reaches the exit.

So what do you do?

You pick one.

```text
Start
  |
 Left
```

Keep walking.

If you find the exit:

```text
Success
```

If you hit a wall:

```text
Dead End
```

then:

```text
Walk Back
```

to the last intersection.

Try another path.

This is literally backtracking.

---

# One-Line Definition

```text
Try a choice.
If it fails, undo it.
Try the next choice.
```

---

# Why Recursion Fits Naturally

Think about the maze.

```text
Start
 |
 A
 |
 B
 |
 C
```

At C:

```text
No solution
```

So return to B.

At B:

```text
Another path exists
```

Try it.

That's exactly what recursion already does.

```cpp
choose();
solve();
undo();
```

The call stack automatically remembers where to come back.

---

# Backtracking = DFS + Undo

This is the most important observation.

When you later learn:

- Trees
    
- Graphs
    
- DFS
    

You'll realize:

```text
Backtracking
=
Depth First Search
+
Undoing Choices
```

# The Universal Pattern

Every backtracking problem is secretly:

```text
Current State
      |
Possible Choices
      |
Try One Choice
      |
Move Forward
      |
If Fail
      |
Undo Choice
      |
Try Next Choice
```

# Permutation Example

Letters:

```text
A B C
```

Goal:

```text
ABC
ACB
BAC
BCA
CAB
CBA
```

---

Visualize:

```text
             ""
        /     |     \
       A      B      C
```

Take A:

```text
A
```

Remaining:

```text
B C
```

Take B:

```text
AB
```

Remaining:

```text
C
```

Take C:

```text
ABC
```

Solution found.

Now:

```text
Remove C
```

Backtrack.

```text
AB
```

No choices.

Remove B.

```text
A
```

Try C.

```text
AC
```

Then B.

```text
ACB
```

---

Notice:

```text
Take
Explore
Undo
Take Another
```

That's backtracking.

---

# The Secret Ingredient: Bounding

Suppose:

```text
AB is not allowed
```

Then:

```text
A
|
AB
```

Immediately stop.

Don't explore below it.

Why?

Because every answer below:

```text
AB_
```

is already invalid.

This is called:

```text
Bounding
```

or

```text
Pruning
```

---

# Why Backtracking Is Powerful

Without pruning:

```text
Try Everything
```

With pruning:

```text
Ignore obviously wrong branches
```

Example:

```text
1000 possibilities
```

Maybe only:

```text
50
```

need to be explored.

Huge speedup.

# Mental Formula

```text
Backtracking
=
Decision Tree Traversal
```

Every problem secretly becomes:

```text
                   Start
                /    |    \
              C1     C2    C3
             / \     / \
            ...
```

The entire game is:

```text
Explore a branch
↓
Fail?
↓
Return
↓
Explore another branch
```

---

# The 4-Step Template

Whenever you solve a backtracking problem, think:

```text
1. What is my current state?

2. What choices can I make?

3. What is the bounding condition?

4. How do I undo my choice?
```

# Ultimate Memory Trick

Imagine a hacker trying passwords:

```text
Try Password
      ↓
Wrong
      ↓
Undo
      ↓
Try Next Password
      ↓
Wrong
      ↓
Undo
      ↓
Try Next
```

That's brute force.

Now add:

```text
"This password is impossible"
```

and skip whole groups of passwords.

That's:

```text
Backtracking
=
Brute Force + Intelligent Pruning
```

And that's the core idea behind:

- Permutations
    
- Combinations
    
- N Queens
    
- Sudoku
    
- Rat in a Maze
    
- Graph Coloring
    
- Word Search
    

All of them are the same pattern wearing different clothes.
