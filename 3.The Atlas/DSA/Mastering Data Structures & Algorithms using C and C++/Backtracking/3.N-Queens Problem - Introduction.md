# What is N-Queens?

Given:

```
N × N ChessboardN Queens
```

Place all queens such that:

```
No two queens attack each other
```

---

# Queen's Movement

A queen can move:

### Horizontally

```
← → 
```

### Vertically

```
↑ ↓
```

### Main Diagonal

```
\
```

### Cross Diagonal

```
/
```

---

# When Are Two Queens Attacking?

Two queens attack each other if they are in:

```
Same RowSame ColumnSame DiagonalSame Cross Diagonal
```

---

# Goal

Place:

```
N Queens
```

on

```
N × N Board
```

such that:

```
No attacks occur.
```

---

# Why Backtracking?

While placing queens:

```
Place Queen 1Place Queen 2Place Queen 3...
```

At some point:

```
No safe position available
```

Then:

```
Remove previous queenTry another position
```

This is:

```
Backtracking
```

---

# Bounding Condition

A queen can be placed only if:

```
Safe RowSafe ColumnSafe DiagonalSafe Cross Diagonal
```

Otherwise:

```
Reject position
```

This is the bounding condition.

---

# Backtracking Idea

```
Place Queen      ↓Safe ?      ↓Yes → Place Next Queen      ↓No      ↓Try Another Position
```

If no position works:

```
Go BackMove Previous QueenContinue
```

---

# Mental Model

Imagine placing queens row by row:

```
Row 1 → Queen 1Row 2 → Queen 2Row 3 → Queen 3...
```

If Row 6 has no valid position:

```
Backtrack to Row 5Move Queen 5Try again
```

---

# Generic Algorithm

```
For every row    Try every column    If position is safe        Place Queen        Solve next row        Remove Queen (Backtrack)
```

---

# Complexity

Backtracking explores many possibilities.

Worst case:

```
O(N!)
```

because we try many arrangements.
