Many students confuse these terms:

- Loop
- Recursion
- Brute Force
- Divide & Conquer
- Dynamic Programming (DP)
- Backtracking

They are **not the same thing**.

Think of them as belonging to different categories:

|Category|Examples|
|---|---|
|Programming Features|Loops|
|Mathematical Concepts|Recursion|
|Problem Solving Approach|Brute Force|
|Problem Solving Strategies|Divide & Conquer, Dynamic Programming, Backtracking|

# 2. Loops vs Recursion

## Loop

A loop is a **feature of programming languages**.

Examples:

```cpp
for(int i=0;i<5;i++)
{   
 cout<<i;
}
```

Programming languages provide:

- [[Variables]]
- [[Data Types]]
- [[Operators]]
- [[Conditions]]
- [[Loops]]

Loop is just another programming feature.

---

## Recursion

Recursion belongs to **mathematics**, not programming.

It existed long before computers were invented.

Example:

```cpp
int fact(int n)
{    
    if(n==0)       
        return 1;    
    return n * fact(n-1);
    
}
```

Mathematicians used recursion centuries before programming languages existed.

---

# 3. Purpose of Loop and Recursion

Both are used for:

## Repetition

### Loop

```cpp
for(int i=1;i<=10;i++)
{  
   cout<<i;
}
```

Repeats statements.

---

### Recursion

```cpp
void print(int n)
{   
   if(n==0)       
    return;    
    cout<<n<<" ";   
    print(n-1);}
```

Also repeats work.

---

# 4. Major Difference

## Loop

Has only one phase:

```
12345Stop
```

Just moves forward.

---

## Recursion

Has two phases:

### Descending Phase

Function keeps calling itself.

```
fun(5)  fun(4)    fun(3)      fun(2)        fun(1)
```

---

### Ascending Phase

Returns back.

```
fun(1)↑fun(2)↑fun(3)↑fun(4)↑fun(5)
```

This is why recursion is powerful.

# 6. Space Complexity

## Loop

Uses fixed memory.

```
for(...){}
```

Space:

```
O(1)
```

(Constant Space)

---

## Recursion

Uses stack memory.

Each call creates:

```
Activation Record
```

Stored inside the call stack.

```
main |fact(5) |fact(4) |fact(3) |fact(2) |fact(1)
```

Space:

```
O(n)
```

---

## Why Recursion is Expensive?

Because stack frames are created.

Too many calls:

```
Stack Overflow
```

may happen.

---

# 7. Historical Fact

Early programming languages:

- FORTRAN
- BASIC

did not support recursion.

Why?

Computers originally supported loops easily.

Recursion required:

```
Function Call Stack
```

which came later.

Eventually languages added recursion support because many mathematical solutions were already written recursively.

# 8. Brute Force

## What is Brute Force?

Brute Force = Try every possibility.

---

### Example

A door has 100 keys.

You don't know the correct key.

Try:

```
Key 1Key 2Key 3...Key 100
```

until it opens.

This is brute force.

---

### Password Cracking

```
abc...aaabac...
```

Trying every possibility.

Brute Force = Exhaustive Search.

---

# Important

Brute Force is NOT recursion.

Brute Force is NOT loops.

Brute Force is an approach.

It can be implemented using:

- Recursion
- Loops

# 9. Divide and Conquer

A strategy for solving large problems.

---

## Core Idea

```
Large Problem     
      ↓
Break into Small Problems     
       ↓
Solve Small Problems      
       ↓
Combine Results     
       ↓
Final Answer
```

---

## Examples

### Binary Search

```
Search 1000 elements
     ↓
Search half
     ↓
Search quarter
     ↓
Search eighth
```

---

### Merge Sort

```
Array
  ↓
Split
  ↓
Split
  ↓
Split↓Merge
```

---

### Quick Sort

Also Divide & Conquer.

---

## Key Idea

```
DivideSolveCombine
```

---

# 10. Dynamic Programming (DP)

## Purpose

Used for:

```
Optimization Problems
```

Finding:

- Maximum
- Minimum
- Best

solution.

---

## Core Idea

Try possible solutions but avoid repeated work.

---

### Example

Fibonacci

Recursive:

```
fib(5)
```

computes:

```
fib(3)fib(3)fib(3)
```

many times.

Wasteful.

---

### DP Solution

Store results.

```
dp[n]
```

Reuse later.

This is:

```
Memoization
```

or

```
Tabulation
```

---

## Goal

```
Optimal Solution+Avoid Repeated Work
```

---

# DP Flow

```
Recursion      ↓Memoization      ↓Tabulation      ↓Optimized Solution
```

---

# 11. Backtracking

Backtracking solves problems where:

```
Multiple Valid Solutions Exist
```

and we want:

```
All Solutions
```

---

## Core Idea

Choose  
↓  
Explore  
↓  
Undo Choice  
↓  
Try Another Choice

---

## Example

Generate subsets

```
[][1][2][3][1,2][1,3][2,3][1,2,3]
```

---

## Example

N-Queens

```
Place Queen↓Valid?↓Yes → ContinueNo → Backtrack
```

---

## Example

Sudoku

```
Try number↓Works?↓Yes → ContinueNo → Remove
```

---

# Backtracking Internally Uses

```
Brute Force+Pruning
```

Meaning:

Try possibilities but stop exploring bad paths early.
