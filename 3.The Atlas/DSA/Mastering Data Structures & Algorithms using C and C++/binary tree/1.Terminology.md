# Trees & Binary Trees — Quick Notes (Interview + DSA Revision)

## 1. What is a Tree?

A **Tree** is a collection of **nodes (vertices)** connected by **edges**.

Example:

```text
        A
      / | \
     B  C  D
    / \
   E   F
```

### Important Property

If a tree has **n nodes**, then:

```text
Number of edges = n - 1
```

Reason:

- Every node except the root has exactly one incoming edge.
    

---

# 2. Basic Terminology

## Root Node

The topmost node.

```text
    A
```

Here:

```text
A = Root
```

---

## Parent and Child

If a node is directly connected below another node:

```text
    B
   / \
  E   F
```

- B → Parent
    
- E, F → Children
    

---

## Siblings

Nodes having the same parent.

```text
    B
   / \
  E   F
```

E and F are siblings.

---

## Descendants

All nodes below a node.

```text
      B
     / \
    E   F
       / \
      J   K
```

Descendants of B:

```text
E, F, J, K
```

---

## Ancestors

All nodes from a node to the root.

```text
      A
      |
      B
      |
      F
      |
      J
```

Ancestors of J:

```text
F, B, A
```

---

# 3. Degree

## Degree of a Node

Number of direct children.

Example:

```text
      D
    / | \
   G  H  I
```

Degree(D):

```text
3
```

---

### Examples

```text
Degree(N) = 0
Degree(J) = 1
Degree(F) = 2
Degree(D) = 3
```

---

## Degree of a Tree

Maximum degree among all nodes.

Example:

```text
Maximum children any node has = 3
```

Then:

```text
Degree of Tree = 3
```

---

# 4. Leaf and Internal Nodes

## Leaf Node (External Node)

Node with degree 0.

```text
E, G, K, N
```

are leaf nodes.

---

## Internal Node (Non-Leaf Node)

Degree > 0

```text
A, B, D, F
```

are internal nodes.

---

# 5. Levels

Level counting starts from **1**.

```text
        A       ← Level 1
      /   \
     B     C    ← Level 2
    / \
   D   E        ← Level 3
```

Formula:

```text
Root = Level 1
```

---

# 6. Height

Height counting starts from **0**.

Count **edges**, not nodes.

```text
        A       ← Height 0
      /   \
     B     C    ← Height 1
    / \
   D   E        ← Height 2
```

### Difference

|Level|Height|
|---|---|
|Counts nodes|Counts edges|
|Starts from 1|Starts from 0|

---

# 7. Subtree

A node along with all its descendants.

```text
      B
     / \
    E   F
```

This entire portion is a subtree.

---

# 8. Forest

A collection of trees.

Example:

```text
Tree1      Tree2      Tree3

  B          C          D
 / \
E   F
```

Multiple separate trees = Forest.

---

# 9. Binary Tree

A tree where each node can have **at most 2 children**.

```text
        A
       / \
      B   C
     / \
    D   E
```

---

## Children Names

```text
      A
     / \
    B   C
```

- B → Left Child
    
- C → Right Child
    

---

## Binary Tree Rule

Each node can have:

```text
0 child
1 child
2 children
```

But NOT:

```text
3 or more children
```

---

# 10. Valid Binary Trees

### Full Node

```text
    A
   / \
  B   C
```

Valid ✅

---

### One Child

```text
    A
   /
  B
```

Valid ✅

---

### No Child

```text
A
```

Valid ✅

---

# 11. Invalid Binary Tree

```text
      A
    / | \
   B  C  D
```

3 children ❌

Not a Binary Tree.

---

# 12. Left Skewed Binary Tree

Every node has only left child.

```text
      A
     /
    B
   /
  C
 /
D
```

Looks like a linked list.

---

# 13. Right Skewed Binary Tree

Every node has only right child.

```text
A
 \
  B
   \
    C
     \
      D
```

### Pattern to Remember

```text
Linked List
     ↓
Skewed Tree
     ↓
Binary Tree
     ↓
BST
     ↓
AVL / Red Black Tree
```

All of these are connected structures made of **nodes and links**. Trees are essentially a generalization of linked lists, and graphs are a generalization of trees. This mental model will help a lot when you reach BSTs, AVL Trees, and Graphs.
