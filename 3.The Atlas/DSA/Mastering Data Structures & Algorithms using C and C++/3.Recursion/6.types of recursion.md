# 1. What is Recursion?

Recursion is a technique where a function solves a problem by calling itself on a smaller version of the same problem.

### General Structure

```cpp
void fun(int n)
{
    if(n == 0)      // Base Case
        return;

    // Processing

    fun(n-1);       // Recursive Call
}
```

Every recursive function must have:

### 1. Base Case

Condition that stops recursion.

```cpp
if(n==0)
    return;
```

Without it:

```cpp
fun()
{
    fun();
}
```

This creates infinite recursion and eventually causes:

```text
Stack Overflow
```

### 2. Recursive Case

The function calls itself.

```cpp
fun(n-1);
```

Consider:

```cpp
void fun(int n)
{
    if(n==0)
        return;
```

Call:

```cpp
fun(3);
```

```text
fun(3)
```

Stack:

```text
| fun(3) |
```

---

#### Step 2

```text
fun(3)
  |
  ---> fun(2)
```

Stack:

```text

| fun(3) |
```

---

#### Step 3

```text
fun(2)
  |
  ---> fun(1)
```

Stack:

```text
| fun(1) |
| fun(2) |

```

---

#### Step 4

```text
fun(1)
  |
  ---> fun(0)
```

Stack:

```text
| fun(0) |
| fun(1) |
| fun(2) |

```

---

#### Base Case Reached

```cpp
if(n==0)
    return;
```

Now stack starts popping.

```text
fun(0) removed
fun(1) removed
fun(2) removed
fun(3) removed
```

# Types of Recursion

There are four major types:

```text
1. Tail Recursion
2. Head Recursion
3. Tree Recursion
4. Indirect Recursion
```

---

# 1. Tail Recursion

## Definition

A recursive function is called Tail Recursive if the recursive call is the last operation performed by the function.

Nothing remains after the recursive call returns.

---

## Structure

```cpp
void fun(int n)
{
    if(n==0)
        return;

    cout<<n<<" ";

    fun(n-1);
}
```

Notice:

```cpp
fun(n-1);
```

is the final statement.

---

## Dry Run

```cpp
fun(3);
```

### Calls

```text
fun(3)
fun(2)
fun(1)
fun(0)
```

### Output Produced

```text
3 2 1
```

---

## Stack Visualization

```text
fun(3)
|
fun(2)
|
fun(1)
|
fun(0)
```

Then returns immediately.

---

## Characteristics

### One Recursive Call

```cpp
fun(n-1);
```

only once.

---

### Work Before Call

```cpp
cout<<n;
fun(n-1);
```

Work happens first.

---

### Easily Converted to Loop

Tail recursion behaves similarly to iteration.

Example:

```cpp
void fun(int n)
{
    while(n>0)
    {
        cout<<n<<" ";
        n--;
    }
}
```

---

## Advantages

- Simple
    
- Less overhead
    
- Similar to loops
    
- Can be optimized by compiler
    

---

## Disadvantages

- Still consumes stack in C++
    

---

# 2. Head Recursion

## Definition

The recursive call occurs before any processing.

Work is done after returning.

---

## Structure

```cpp
void fun(int n)
{
    if(n==0)
        return;

    fun(n-1);

    cout<<n<<" ";
}
```

---

## Dry Run

```cpp
fun(3);
```

Calls:

```text
fun(3)
fun(2)
fun(1)
fun(0)
```

No output yet.

---

### Returning Phase

```text
fun(1) prints 1
fun(2) prints 2
fun(3) prints 3
```

Output:

```text
1 2 3
```

---

## Internal Understanding

### Going Down

```text
3
2
1
0
```

### Coming Up

```text
1
2
3
```

---

## Stack Visualization

```text
Push Push Push Push
      ↓
Base Case
      ↓
Pop Pop Pop Pop
```

Actual work happens while popping.

---

## Characteristics

### One Recursive Call

```cpp
fun(n-1);
```

---

### Work After Call

```cpp
fun(n-1);

cout<<n;
```

---

## Applications

Useful when:

- Reversing operations
    
- Tree traversals
    
- Backtracking
    

---

# Difference Between Tail and Head Recursion

|Tail|Head|
|---|---|
|Work before recursion|Work after recursion|
|Output: 3 2 1|Output: 1 2 3|
|Similar to loops|Uses return path|
|Easier optimization|Less optimization|

---

# 3. Tree Recursion

## Definition

A function makes more than one recursive call.

---

## Structure

```cpp
void fun(int n)
{
    if(n<=0)
        return;

    cout<<n<<" ";

    fun(n-1);
    fun(n-1);
}
```

Notice:

```cpp
fun(n-1);
fun(n-1);
```

Two recursive calls.

---

# Why Called Tree Recursion?

Because execution forms a tree.

---

## Dry Run

```cpp
fun(3)
```

Creates:

```text
          3
        /   \
       2     2
      / \   / \
     1  1 1   1
```

---

### Function Calls

```text
fun(3)

fun(2)
fun(2)

fun(1)
fun(1)
fun(1)
fun(1)
```

Total calls increase rapidly.

---

## Growth Pattern

```text
n=1 → 1 call
n=2 → 3 calls
n=3 → 7 calls
n=4 → 15 calls
```

Approximately:

```text
O(2^n)
```

---

## Famous Example: Fibonacci

```cpp
int fib(int n)
{
    if(n<=1)
        return n;

    return fib(n-1)+fib(n-2);
}
```

---

### Fibonacci Tree

```text
fib(5)
├─ fib(4)
│  ├─ fib(3)
│  └─ fib(2)
└─ fib(3)
   ├─ fib(2)
   └─ fib(1)
```

Notice:

```text
fib(3)
fib(2)
```

are computed repeatedly.

This is why naive Fibonacci is slow.

---

## Characteristics

### Multiple Recursive Calls

```cpp
fun(n-1);
fun(n-1);
```

---

### Creates Tree Structure

Useful for:

- Binary Trees
    
- DFS
    
- Backtracking
    
- Game Trees
    

---

## Advantages

Natural representation of branching problems.

---

## Disadvantages

Very expensive.

Can cause:

```text
Exponential Time Complexity
```

---

# 4. Indirect Recursion

## Definition

A function does not call itself directly.

Instead multiple functions call each other.

---

## Structure

```cpp
A()
{
    B();
}

B()
{
    A();
}
```

---

# Example

```cpp
void B(int n);

void A(int n)
{
    if(n<=0)
        return;

    cout<<"A ";

    B(n-1);
}

void B(int n)
{
    if(n<=0)
        return;

    cout<<"B ";

    A(n-1);
}
```

---

## Dry Run

```cpp
A(3)
```

Execution:

```text
A(3)
↓
B(2)
↓
A(1)
↓
B(0)
```

Output:

```text
A B A
```

---

## Call Diagram

```text
A
↓
B
↓
A
↓
B
```

---

## Characteristics

### Multiple Functions

Instead of:

```cpp
A → A
```

we get:

```text
A → B → A → B
```

---

## Applications

Used in:

- Parsers
    
- Compilers
    
- State Machines
    
- Graph Algorithms
    

---

# Complete Comparison

|Feature|Tail|Head|Tree|Indirect|
|---|---|---|---|---|
|Recursive Calls|1|1|More than 1|Through another function|
|Work Position|Before call|After call|Both possible|Depends|
|Structure|Linear|Linear|Branching|Circular|
|Space Complexity|O(n)|O(n)|O(n)|O(n)|
|Time Complexity|O(n)|O(n)|Often O(2ⁿ)|Depends|

---

# How to Identify Recursion Type in Exam

### Tail Recursion

```cpp
print();
fun(n-1);
```

Ask:

> Is recursive call the last statement?

✅ Tail Recursion

---

### Head Recursion

```cpp
fun(n-1);
print();
```

Ask:

> Is work done after recursion returns?

✅ Head Recursion

---

### Tree Recursion

```cpp
fun(n-1);
fun(n-1);
```

Ask:

> More than one recursive call?

✅ Tree Recursion

---

### Indirect Recursion

```cpp
A()
{
   B();
}

B()
{
   A();
}
```

Ask:

> Is another function involved before coming back?

✅ Indirect Recursion

---

# The Big Picture

Think of recursion as different shapes:

```text
Tail Recursion

3
|
2
|
1
|
0
```

```text
Head Recursion

0
↑
1
↑
2
↑
3
```

```text
Tree Recursion

        3
      /   \
     2     2
    / \   / \
   1  1 1   1
```

```text
Indirect Recursion

A ↔ B ↔ A ↔ B
```

If you deeply understand these four patterns, you'll find it much easier to learn **binary trees, DFS, backtracking, divide-and-conquer, dynamic programming, graph traversal, and compiler design**, because they are all built on the same recursive ideas.
