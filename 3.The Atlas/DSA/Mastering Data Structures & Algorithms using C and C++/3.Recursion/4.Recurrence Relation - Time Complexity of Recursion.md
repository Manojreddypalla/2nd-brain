## Recurrence Relation – Time Complexity of Recursion

Think like this:

A recursive function does **some work now**

→ then calls a **smaller version of itself**.

So total time becomes:

```
Current Work + Smaller Recursive Work
```

That is called a **Recurrence Relation**.

---

# 1. Simple Example

```cpp
#include <iostream>
using namespace std;

void fun(int n)
{
    if(n > 0)
    {
        cout << n << endl;   // 1 unit work
        fun(n - 1);          // recursive call
    }
}

int main()
{
    fun(5);
}
```

---

# What Happens Internally

```
fun(5)
 └── fun(4)
      └── fun(3)
           └── fun(2)
                └── fun(1)
                     └── fun(0)
```

Each call:

- prints once
- calls smaller recursion

So work repeats `n` times.

---

# Recurrence Relation

For every call:

```
T(n) = T(n-1) + 1
```

Meaning:

```
Total Time
=
Smaller Recursive Time
+
Current Work
```

Base case:

```
T(0) = 1
```

---

# Solving It

```
T(n) = T(n-1) + 1
T(n-1) = T(n-2) + 1
T(n-2) = T(n-3) + 1
```

Substitute repeatedly:

```
T(n) = T(n-2) + 2
T(n) = T(n-3) + 3
...
T(n) = T(0) + n
```

Since:

```
T(0) = 1
```

Final:

```
T(n) = n + 1
```

So:

```
Time Complexity = O(n)
```

---

# 2. Factorial Example

```cpp
#include <iostream>
using namespace std;

int fact(int n)
{
    if(n == 0)
        return 1;

    return n * fact(n - 1);
}

int main()
{
    cout << fact(5);
}
```

---

# Recurrence Relation

```
T(n) = T(n-1) + 1
```

Why?

Because:

- multiplication → 1 work
- recursive call → T(n-1)

So again:

```
O(n)
```

---

# 3. Fibonacci Example

```cpp
#include <iostream>
using namespace std;

int fib(int n)
{
    if(n <= 1)
        return n;

    return fib(n-1) + fib(n-2);
}

int main()
{
    cout << fib(6);
}
```

---

# Internal Recursion Tree

```
fib(5)
├── fib(4)
│   ├── fib(3)
│   └── fib(2)
└── fib(3)
```

Notice:

```
fib(3) repeats again
```

This duplication causes explosion.

---

# Recurrence Relation

```
T(n) = T(n-1) + T(n-2) + 1
```

This grows exponentially.

Final complexity:

```
O(2^n)
```

---

# 4. Binary Search Recursion

```cpp
#include <iostream>
using namespace std;

int binarySearch(int arr[], int low, int high, int key)
{
    if(low > high)
        return -1;

    int mid = (low + high) / 2;

    if(arr[mid] == key)
        return mid;

    else if(key < arr[mid])
        return binarySearch(arr, low, mid - 1, key);

    else
        return binarySearch(arr, mid + 1, high, key);
}

int main()
{
    int arr[] = {1,2,3,4,5,6,7,8};

    cout << binarySearch(arr,0,7,6);
}
```

---

# Recurrence Relation

Each time array becomes half:

```
T(n) = T(n/2) + 1
```

Final complexity:

```
O(log n)
```

---

# Pattern Recognition

## If recursion reduces by 1

```
T(n) = T(n-1) + c
```

→ Linear

```
O(n)
```

---

## If recursion halves input

```
T(n) = T(n/2) + c
```

→ Logarithmic

```
O(log n)
```

---

## If recursion branches twice

```
T(n) = T(n-1) + T(n-2)
```

→ Exponential

```
O(2^n)
```

---

# Core Intuition

Recursion complexity depends on:

1. **How many recursive calls**
2. **How much input shrinks**
3. **Extra work done per call**

That’s the real mental model.

---

Small recursion tree:

```
One branch + shrink by 1  -> O(n)

One branch + half input   -> O(log n)

Two branches              -> Exponential
```
