# How Recursion Uses Stack — Short Notes

When a recursive function calls itself:

```cpp
fun(3);
```

the system stores every function call inside a **Stack Memory**.

---

# What gets stored in Stack?

Each function call creates an:

```
Activation Record / Stack Frame
```

It stores:

- Parameters
- Local variables
- Return address

---

# Example

```cpp
#include <iostream>
using namespace std;

void fun(int n)
{
    if(n > 0)
    {
        cout << n << " ";
        fun(n - 1);
    }
}

int main()
{
    fun(3);
}
```

---

# Internally Stack Becomes

## Step 1

```
fun(3)
```

Stack:

```
| fun(3) |
```

---

## Step 2

```
fun(2)
```

Stack:

```
| fun(2) |
| fun(3) |
```

---

## Step 3

```
fun(1)
```

Stack:

```
| fun(1) |
| fun(2) |
| fun(3) |
```

---

## Step 4

```
fun(0)
```

Stack:

```
| fun(0) |
| fun(1) |
| fun(2) |
| fun(3) |
```

---

# Returning Phase

Now `n == 0`

So function returns.

Stack pops backward:

```
fun(0) removed
fun(1) removed
fun(2) removed
fun(3) removed
```

---

# Important Idea

Recursion works because stack remembers:

```
Where function stopped
What variables existed
Where to return
```

That’s why recursion can continue correctly after returning.

---

# Visual Flow

```
CALLING PHASE
fun(3)
  fun(2)
    fun(1)
      fun(0)

RETURNING PHASE
      return
    return
  return
return
```

---

# Memory Usage

If recursion depth is `n`:

```
Space Complexity = O(n)
```

Because stack stores `n` function calls.

---

# Important Rule

```
Every recursive call pushes into stack.
Every return pops from stack.
```

---

# Why Stack Overflow Happens

If recursion never stops:

```cpp
fun()
{
    fun();
}
```

Stack keeps growing forever.

Eventually:

```
Stack Overflow
```

# Tiny Debug Code

```cpp
#include <iostream>
using namespace std;

void fun(int n)
{
    cout << "Entered: " << n << endl;

    if(n > 0)
    {
        fun(n - 1);
    }

    cout << "Returning: " << n << endl;
}

int main()
{
    fun(3);
}
```

## Output

```
Entered: 3
Entered: 2
Entered: 1
Entered: 0
Returning: 0
Returning: 1
Returning: 2
Returning: 3
```
