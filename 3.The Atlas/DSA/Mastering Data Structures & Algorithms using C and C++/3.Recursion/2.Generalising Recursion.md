# Recursion Important Notes

Recursion = a function calling itself.

```
fun()
{
fun();
}
```

But recursion must have a **base condition** to stop.

---

# Core Structure of Recursion

```
voidfun(intn)
{
if(n>0)// Base condition
    {
// work

fun(n-1);// Recursive call

// work
    }
}
```

---

# Most Important Idea

Recursion has **2 phases**:

1. Calling Phase (Ascending)
2. Returning Phase (Descending)

---

# Calling Phase

Happens BEFORE recursive call.

```
cout<<n;
fun(n-1);
```

This executes while going deeper.

---

# Returning Phase

Happens AFTER recursive call.

```
fun(n-1);
cout<<n;
```

This executes while stack returns back.

---

# Visual Understanding

Suppose:

```
fun(3)
```

Calls become:

```
fun(3)
  fun(2)
    fun(1)
      fun(0)
```

Then returns backward:

```
fun(0) returns
fun(1) returns
fun(2) returns
fun(3) returns
```

---

```
#include<iostream>

# Both Phases Together

```
#include<iostream>
usingnamespacestd;

voidfun(intn)
{
if(n>0)
    {
cout<<"Calling: "<<n<<endl;

fun(n-1);

cout<<"Returning: "<<n<<endl;
    }
}

intmain()
{
fun(3);
}
```

## Output

```
Calling: 3
Calling: 2
Calling: 1
Returning: 1
Returning: 2
Returning: 3
```

This is the BEST code to understand recursion deeply.

# Stack Understanding

Every recursive call creates a new stack frame.

Each frame stores:

- Parameters
- Local variables
- Return address

Example:

```
fun(3)
```

Internally stack becomes:

```
| fun(3) |
| fun(2) |
| fun(1) |
| fun(0) |
```

Then stack pops backward.

---

# Infinite Recursion

Wrong:

```
voidfun()
{
fun();
}
```

This never stops.

Eventually:

```
Stack Overflow
```

---

# Base Condition

The stopping condition.

```
if(n==0)
return;
```

Without this recursion is dangerous.

---

# Recursion vs Loop

|Loop|Recursion|
|---|---|
|Only ascending|Ascending + descending|
|Less memory|Uses stack|
|Faster usually|Cleaner for trees/graphs|
|Iteration|Function calls|

# Factorial Example

## Logic

```
5! = 5 × 4 × 3 × 2 × 1
```

Recursive thinking:

```
5! = 5 × 4!
```

---

## Code

```
#include<iostream>
usingnamespacestd;

intfact(intn)
{
if(n==0)
return1;

returnn*fact(n-1);
}

intmain()
{
cout<<fact(5);
}
```

## Output

```
120
```

{
if(n<=1)
returnn;

returnfib(n-1)+fib(n-2);

```

# Recursive Thinking Pattern

Always ask:

## 1. What is the smallest problem?

Usually:

```
n==0
```

---

## 2. How to reduce problem?

```
fun(n-1)
```

---

## 3. What should current function do?

Before call?

After call?

Or both?
