### Simple LeetCode Problem Solving Process

1. **Read the problem twice**
    
    - Understand input and output.
        
    - Check constraints and edge cases.
        
2. **Think of simple solutions first**
    
    - Don't jump into coding.
        
    - Ask: "What's the most basic way to solve this?"
        
3. **Compare different approaches**
    
    - Brute Force
        
    - Better Solution
        
    - Optimal Solution
        
4. **Analyze complexity**
    
    - Time Complexity: How fast?
        
    - Space Complexity: How much memory?
        
5. **Draw an example**
    
    - Use paper.
        
    - Trace arrays, trees, graphs, pointers, etc.
        
    - Look for patterns.
        
6. **Identify the pattern**
    
    - Hash Map?
        
    - Two Pointers?
        
    - Sliding Window?
        
    - Binary Search?
        
    - DFS/BFS?
        
    - DP?
        
    - Greedy?
        
7. **Write the algorithm in words**
    
    - List the steps before coding.
        
8. **Dry run manually**
    
    - Walk through a sample input.
        
    - Verify your logic.
        
9. **Code the solution**
    
    - Convert your algorithm into code.
        
10. **Test edge cases**
    
    - Empty input
        
    - Single element
        
    - Duplicates
        
    - Large inputs
        
    - Negative values
        
11. **Optimize if possible**
    
    - Reduce time complexity.
        
    - Reduce space complexity.
        
    - Simplify code.
        
12. **Study other solutions**
    
    - Learn new tricks.
        
    - Find better patterns.
        
    - Understand different approaches.
