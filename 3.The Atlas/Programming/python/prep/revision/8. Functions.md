# 8. Functions

## Why do functions exist?

Imagine writing this everywhere:

```python
a = 10
b = 20
print(a + b)

x = 5
y = 7
print(x + y)

p = 100
q = 200
print(p + q)
```

You're repeating the same logic.

Instead:

```python
def add(a, b):
    return a + b

print(add(10, 20))
print(add(5, 7))
print(add(100, 200))
```

A function lets you **write logic once and reuse it**.

# Variable-Length Arguments

## *args

Suppose you don't know how many numbers the user will pass.

Call

# Passing Mutable vs Immutable Objects

This is where your earlier question about references becomes important.

### Immutable

```python
def change(x):
    x = 100

a = 10

change(a)

print(a)
```

Output

```
10
```

Why?

A new integer object is created inside the function.

---

### Mutable

```python
def change(lst):
    lst.append(4)

nums = [1,2,3]

change(nums)

print(nums)
```

Output

```
[1,2,3,4]
```

Why?

The function receives a **reference to the same list**, and `append()` modifies that list.

This is one of the most common interview questions.

### Q5

```python
def test(x=[]):
    x.append(1)
    return x

print(test())
print(test())
```

Output

```
[1]
[1, 1]
```

This is a famous interview question.

**Why?**

Default arguments are evaluated **once**, when the function is defined. The same list object is reused on subsequent calls. The safer pattern is:

```python
def test(x=None):
    if x is None:
        x = []
    x.append(1)
    return x
```
