# 10. Exception Handling

## What is an Exception?

**Definition:** An exception is a runtime error that interrupts the normal flow of a program.

Example:

```python
print(10 / 0)
```

Output:

```text
ZeroDivisionError
```

---

# 1. `try` and `except` ⭐⭐⭐⭐⭐

```python
try:
    x = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero")
```

---

# 2. Multiple Exceptions

```python
try:
    x = int("abc")
except ValueError:
    print("Invalid Number")
except ZeroDivisionError:
    print("Division Error")
```

---

# 3. `else`

Runs only if **no exception** occurs.

```python
try:
    print(10 / 2)
except:
    print("Error")
else:
    print("Success")
```

Output

```text
5.0
Success
```

---

# 4. `finally` ⭐⭐⭐⭐⭐

Always executes.

```python
try:
    print(10 / 0)
except:
    print("Error")
finally:
    print("Done")
```

Output

```text
Error
Done
```

Use it for closing files or database connections.

---

# 5. `raise`

Raise your own exception.

```python
age = -5

if age < 0:
    raise ValueError("Age cannot be negative")
```

---

# Common Built-in Exceptions

|Exception|Cause|
|---|---|
|`ZeroDivisionError`|Divide by zero|
|`ValueError`|Wrong value/type conversion|
|`TypeError`|Wrong data type|
|`IndexError`|List index out of range|
|`KeyError`|Dictionary key not found|
|`NameError`|Variable not defined|
|`FileNotFoundError`|File doesn't exist|

# Remember this order

```python
try:
    # risky code

except:
    # handle error

else:
    # runs if no error

finally:
    # always runs
```
