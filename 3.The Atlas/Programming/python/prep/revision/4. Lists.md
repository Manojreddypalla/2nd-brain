# 4. Lists

## What is a List?

A **list** is an **ordered, mutable collection** of objects.

Think of it as Python's version of a **dynamic array**.

### C++

```cpp
vector<int> nums = {10,20,30};
```

### Python

```python
nums = [10,20,30]
```

Very similar.

---

# Internal Memory

```python
nums = [10,20,30]
```

Memory (conceptually)

```text
nums
 │
 ▼
+----------------------------+
| 10 | 20 | 30 |
+----------------------------+
  0    1    2
```

Unlike arrays in C, **lists store references to objects**, not raw values.

```python
nums = [10, "Hello", True]
```

Memory:

```text
nums
 │
 ▼
+----+----+----+
| •  | •  | •  |
+----+----+----+
 |     |     |
 ▼     ▼     ▼
10   "Hello" True
```

This is why Python lists can contain **different data types**.

# Copy

This is a favorite interview question.

## Wrong

```python
a = [1,2,3]

b = a
```

Memory

```text
a ----\
       \
        ----> [1,2,3]
       /
b ----/
```

Now

```python
b.append(4)
```

Both become

```python
[1,2,3,4]
```

Because both variables refer to the **same list object**.

---

## Correct Copy

```python
b = a.copy()
```

or

```python
b = a[:]
```

Memory

```text
a ---> [1,2,3]

b ---> [1,2,3]
```

Two different objects.

# List Comprehension ⭐⭐⭐⭐⭐

One of Python's most powerful features.

Instead of

```python
nums = []

for i in range(5):
    nums.append(i)
```

Use

```python
nums = [i for i in range(5)]
```

↓

```python
[0,1,2,3,4]
```

---

Even numbers

```python
nums = [i for i in range(10) if i%2==0]
```

↓

```python
[0,2,4,6,8]
```

Interviewers love this.

# Pattern Recognition

When you see a list question, ask yourself:

1. Is the list **being modified** or is a **new list** being created?
    
2. Is it a **reference assignment (`b = a`)** or a **copy (`b = a.copy()`)**?
    
3. Is the operation **append** (one element) or **extend** (many elements)?
    
4. Is the question about **time complexity**, **mutability**, or **list comprehension**?
    

---

## Connection to C++

|C++|Python|
|---|---|
|`vector<int>`|`list`|
|`push_back()`|`append()`|
|`size()`|`len()`|
|`sort()`|`sort()`|
|`reverse()`|`reverse()`|
|Range-based `for`|`for x in nums`|

The biggest conceptual difference is that **Python lists store references to objects**, allowing them to hold mixed types, whereas a `vector<int>` stores integers directly.
