# 7. Dictionaries

## First Question

Why do dictionaries exist?

Suppose you have student data.

Without a dictionary:

```python

    ["Retro",22],
    ["Alex",21],
    ["John",23]
]
```

Now someone asks:

> Find Retro's age.

Python has to search.

```
Retro?

No

No

Yes
```

Worst case

```
O(n)
```

Not efficient.

---

Instead

```python
student = {
    "Retro":22,
    "Alex":21,
    "John":23
}
```

Now Python does

```

↓

Directly jumps

↓

22
```

Average

```
O(1)
```

This is why dictionaries exist.

# Dictionary vs Set

This is where many people get confused.

### Set

```
hash(key)

↓

Exists?

Yes / No
```

Stores only **keys** (elements).

Example:

```python
{"apple", "banana", "mango"}
```

---

### Dictionary

```
hash(key)

↓

Find bucket

↓

Return value
```

Stores **key → value** pairs.

Example:

```python
{
    "apple": 50,
    "banana": 30
}
```

---

# The Deep Internal Picture

You've learned:

- **List** → Dynamic array of **references**.
    
- **Tuple** → Fixed (immutable) sequence of **references**.
    
- **Set** → Hash table storing **unique keys**.
    
- **Dictionary** → Hash table storing **key → value** mappings.
    

Notice something interesting:

```
Dictionary
```

is almost like

```
Set
      +
extra value attached to every key
```

That's why they behave so similarly:

- Both use hashing.
    
- Both require hashable keys.
    
- Both have average O(1) lookups.
    
- Both don't allow duplicate keys.

Difference:
