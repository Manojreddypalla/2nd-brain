# Python Data Types

## What is a Data Type?

A **data type** tells Python:

- What kind of value this is.
    
- What operations are allowed on it.
    
- How it should be stored in memory.
    

Think of it like this:

```
Value → Object → Type
```

For example:

```python
x = 10
```

Internally:

```
x
│
▼
+-----------------------+
| int object            |
| value = 10            |
| type = int            |
+-----------------------+
```

Everything in Python is an **object**.

Even this:

```python
5
```

is actually an `int` object.

# Categories of Data Types

## 1. Numeric Types

```python
a = 10          # int
b = 3.14        # float
c = 2 + 3j      # complex
```

### int

Whole numbers

```python
x = 100
```

Operations

```python
+
-
*
/
//
%
**
```

Example

```python
print(10 // 3)
```

Output

```
3
```

Why?

`//` performs **floor division** (it returns the largest integer less than or equal to the result).

---

```python

```

Stores decimal numbers.

Example

```python

```

Output

```
2.5
```

Notice:

`/`

always returns a float.

### complex

Python supports complex numbers directly.

```python

```

Real part

```python

```

Imaginary part



# 7. Dictionary

Probably the most important data structure in Python.

```python
student = {
    "name":"Retro",
    "age":22
}
```

Stores

```
Key → Value
```

```
"name" → "Retro"

"age" → 22
```

Internally it uses a **hash table**, giving average **O(1)** lookup, insertion, and deletion.

Example

```python
student["name"]
```

Output

```
Retro
```

# Mutable vs Immutable (Very Important)

|Mutable|Immutable|
|---|---|
|list|int|
|dict|float|
|set|bool|
|bytearray|string|
||tuple|
||frozenset|

**Rule to remember:**

- Mutable → contents can change.
    
- Immutable → contents cannot change after creation.

# Truthy and Falsy Values

Python treats some values as `False` in conditions.

Falsy values include:

```python
False
None
0
0.0
''
[]
{}
set()
```

Everything else is generally truthy.

Example:

```python
if []:
    print("Hello")
else:
    print("Empty")
```

Output:

```
Empty
```

# Common Interview MCQs

### Q1

```python
print(type(5))
```

**Answer:**

```
<class 'int'>
```

---

### Q2

```python
print(type(5.0))
```

**Answer:**

```
<class 'float'>
```

---

### Q3

```python
print(bool(""))
```

**Answer:**

```
False
```

---

### Q4

```python
print(bool([0]))
```

**Answer:**

```
True
```

Because the list contains an element. An **empty** list (`[]`) is falsy, but a non-empty list is truthy regardless of its contents.

---

### Q5

```python
print(type({}))
```

**Answer:**

```
<class 'dict'>
```

---

### Q6 (Classic)

```python
print(type({1}))
```

**Answer:**

```
<class 'set'>
```

Remember:

- `{}` → empty dictionary
    
- `{1}` → set containing one element
    
- To create an empty set, use:
    

```python
set()
```

## Pattern Recognition

Whenever you see a Python data type question, ask yourself:

1. What is the type?
    
2. Is it mutable or immutable?
    
3. Is it ordered?
    
4. Does it allow duplicates?
    
5. How is it typically accessed? (index, key, hash lookup, etc.)
    

These five questions let you reason through most data-type MCQs instead of memorizing answers.

After this, the natural next topic is **Strings**, because many coding problems and Python interview questions build directly on string operations.
