# What is a String?

A string is simply a **sequence of characters**.

```python
name = "Retro"
```

Memory (conceptually):

```text
        String Object
+---------------------------+
|  R | e | t | r | o |
+---------------------------+
  0   1   2   3   4
```

Unlike C, **Python already knows the string's length**. There is no null character (`'\0'`) that you deal with directly.

---

# Strings are Objects

```python
s = "Hello"
```

Internally:

```text
s
 │
 ▼
+----------------+
| "Hello"        |
| type = str     |
+----------------+
```

Everything you do (`upper()`, `lower()`, slicing) is calling methods on this object.

---

# The Most Important Property

## Strings are Immutable

This is the **#1 interview concept**.

```python
s = "Hello"

s[0] = "Y"
```

Output:

```text
TypeError
```

Why?

Because Python doesn't allow changing individual characters.

Instead, it creates a **new string**.

```python
s = "Y" + s[1:]
```

Now:

```text
Old String

Hello

New String

Yello
```

The old string wasn't modified.

---

# Comparison with C++

### C++

```cpp
string s = "Hello";

s[0] = 'Y';
```

✅ Allowed

---

### Python

```python
s = "Hello"

s[0] = "Y"
```

❌ Error

This difference is asked surprisingly often.

# Indexing

Exactly like arrays.


```text

0  1  2  3  4  5
```

```python
print(s[0])
```

Output

```text

```

---

Negative indexing:

```text

-6 -5 -4 -3 -2 -1
```

```python

```

Output

```text
n
```

Very useful.

# Slicing

One of Python's biggest strengths.

Syntax:

```python
string[start:end:step]
```

Think:

```text
[start , end)
```

End is **not included**.

Example:

```python
s = "Python"

print(s[0:3])
```

Output

```text
Pyt
```

Because

```text
0 1 2
```

---

### Examples

```python
s = "Python"

s[:3]
```

Output

```text
Pyt
```

---

```python
s[2:]
```

Output

```text
thon
```

---

```python
s[-3:]
```

Output

```text
hon
```

---

Reverse string

```python
s[::-1]
```

Output

```text
nohtyP
```

This is one of Python's most famous interview tricks.

Very important.

# Pattern Recognition

When you see a string question, ask yourself:

1. Is the string being **modified**? (Remember: strings are immutable.)
    
2. Is it **indexing** or **slicing**?
    
3. What does the slice syntax `[start:end:step]` produce?
    
4. Is it using a common method like `split()`, `join()`, `replace()`, or `find()`?

## 🔥 Interview Connection

You've solved **string problems in C++** like reversing a string, checking palindromes, and finding substrings.

In Python, the **algorithms stay the same**. Only the syntax becomes shorter.

For example, in C++ you might reverse a string using a loop or `reverse()`:

```cpp
reverse(s.begin(), s.end());
```

In Python:

```python
s[::-1]
```

The thinking hasn't changed—Python simply gives you more expressive tools.
