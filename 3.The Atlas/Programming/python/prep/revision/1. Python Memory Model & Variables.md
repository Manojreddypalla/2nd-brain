# Module 1: Variables & Memory

## First, forget the syntax.

Think about **what a variable really is**.

### In C++

```cpp
int x = 10;
```

Memory looks like:

```
Address
1000 ----> 10
```

`x` directly owns that memory location.

---

### In Python

```python
x = 10
```

Many beginners think this is identical.

It isn't.

Python does:

```
        x
        |
        v
+----------------+
| Integer Object |
|      10        |
+----------------+
```

The variable **doesn't store the value**.

It stores a **reference** to an object.

Think of the variable as a sticky note pointing to a box.

---

## Example

```python
x = 10
y = x
```

Memory:

```
x ----\
       \
        ---> [10]
       /
y ----/
```

Both point to the same integer object.

---

Now:

```python
x = 20
```

Does Python change the object?

No.

Integers are immutable.

It creates another object.

```
x ------> [20]

y ------> [10]
```

This is one of the most common MCQs.

---

# Mutable vs Immutable

This is **extremely important**.

## Immutable

Cannot change.

- int
    
- float
    
- bool
    
- string
    
- tuple
    

Example

```python
x = 5
x = 6
```

The object `5` wasn't modified. `x` now points to a different object.

---

## Mutable

Can change.

- list
    
- dict
    
- set
    

Example

```python
nums = [1,2]

nums.append(3)
```

Memory

Before

```
nums

↓

[1,2]
```

After

```
nums

↓

[1,2,3]
```

Same object.

Only its contents changed.

# Identity vs Equality

Another favorite interview question.

```python
a = [1,2]

b = [1,2]
```

Question:

```
a == b
```

Answer?

```
True
```

Because values are equal.

---

Now

```python
a is b
```

Answer?

```
False
```

Because they are different objects.

Memory

```
a ---> [1,2]

b ---> [1,2]
```

Different locations.

---

Now

```python
a = [1,2]

b = a
```

Memory

```
a -----\

        > [1,2]

b -----/
```

Now

```
a is b
```

True.

# Small Integer Caching (Python MCQ)

Predict:

```python
a = 100
b = 100

print(a is b)
```

Usually

```
True
```

Why?

Python caches small integers (commonly from `-5` to `256` in CPython).

---

Now

```python
a = 1000
b = 1000

print(a is b)
```

This is **implementation-dependent**. In many situations it will be `False`, but you should **not** rely on `is` for comparing numbers. Use `==` to compare values.

**Rule for interviews:**

- `==` → compare values.
    
- `is` → compare object identity.

## Think like an interviewer

Whenever you see Python code, ask yourself:

1. **How many objects are created?**
    
2. **Which variables point to the same object?**
    
3. **Is the object mutable or immutable?**
    
4. **Is the code comparing values (`==`) or identity (`is`)?**
    

If you build this habit, a large class of Python MCQs becomes much easier.
