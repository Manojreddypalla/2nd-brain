# Hacking Lab — Tools & Quick Notes

## WhatWeb

- **Purpose:** Ruby-based web scanner that fingerprints websites & application stack (CMS, WAFs, webcams, etc.). Plugin system (~1,800 plugins).
- **Install:** `sudo apt-get install whatweb -y`
- **Verify / help:** `whatweb -h`
- **Use case:** Recon — identify backend tech (e.g., “This site runs WordPress + nginx + Cloudflare”).
- **Example:** run against a target: `whatweb example.com`

---

## RustScan

- **Purpose:** Very fast port scanner written in Rust (scans many ports quickly).
    
- **Install (Docker):** `sudo docker pull rustscan/rustscan:2.1.1`
    
- **Test run:**
    
    `sudo docker run --network=host -it --rm --name rustscan rustscan/rustscan:2.1.1`
    
- **Note/Warning:** Don’t run against sensitive infra — many concurrent sockets can overwhelm targets.
    
- **Tip:** Alias it for convenience (see “Aliases” below).
    

---

## Nuclei

- **Purpose:** Template-based vulnerability scanner (uses YAML templates). Large community template repo (projectdiscovery / nuclei-templates).
- **Install:** `sudo apt install nuclei -y`
- **Verify / help:** `nuclei -h`
- **Behavior:** On first run it auto-downloads nuclei-templates to `~/nuclei-templates`.
- **Use case:** Automated vulnerability checks using community templates.
- **Example:** `nuclei -u <https://example.com> -t templates/` (template usage varies)

---

## dirsearch

- **Purpose:** Multithreaded web path/directory discovery (find common paths).
- **Install:** `sudo apt install dirsearch -y`
- **Verify / help:** `dirsearch --help`
- **Use case:** Find hidden directories/files (e.g., `/admin`, `/uploads`).

---

## Linux Exploit Suggester 2 (LES2)

- **Purpose:** Suggests kernel/local Linux exploits based on system fingerprint.
    
- **Install:**
    
    ```
    cd ~/tools
    git clone <https://github.com/jondonas/linux-exploit-suggester-2.git>
    
    ```
    
- **Verify/help:**
    
    ```
    cd linux-exploit-suggester-2
    perl linux-exploit-suggester-2.pl -h
    
    ```
    
- **Use case:** Privilege escalation research for a target kernel/version.
    

---

## Gitjacker

- **Purpose:** Extract data from accidentally exposed `.git` directories.
    
- **Prereq:** `jq` (JSON processor) — `sudo apt install jq -y`
    
- **Install (script):**
    
    ```
    cd ~
    curl -s "<https://raw.githubusercontent.com/liamg/gitjacker/master/scripts/install.sh>" | bash
    mv ./bin/gitjacker ~/tools/gitjacker
    rmdir ./bin
    
    ```
    
- **Verify/help:** `~/tools/gitjacker -h`
    
- **Use case:** Recover source/config that leaked via `.git` upload.
    

---

## pwncat

- **Purpose:** Python C2 / post-exploitation shell manager and exploitation platform.
- **Install:** `pip3 install pwncat-cs`
- **Verify/help:** `pwncat-cs -h`
- **Note:** May be installed under `~/.local/bin/pwncat-cs` — run full path if necessary.
- **Use case:** Interact with compromised shells, pivot, post-exploit tooling.

---

## LinEnum

- **Purpose:** Local Linux enumeration & privilege escalation script (bash).
    
- **Install (download):**
    
    ```
    cd ~/tools
    wget <https://raw.githubusercontent.com/rebootuser/LinEnum/master/LinEnum.sh>
    chmod u+x LinEnum.sh
    ./LinEnum.sh -h
    
    ```
    
- **Use case:** Quick host info & privilege escalation checks.
    

---

## unix-privesc-check

- **Purpose:** Shell script to collect host info and find privilege escalation points.
- **Location (Kali):** `/usr/bin/unix-privesc-check`
- **Find it:** `which unix-privesc-check`
- **Optional:** Copy to tools: `cp /usr/bin/unix-privesc-check ~/tools`
- **Install if missing:** `apt-get install unix-privesc-check -y`
- **Verify/help:** `unix-privesc-check -h`
- **Use case:** Fast privilege escalation reconnaissance.

## Assigning Aliases (make tools easy to run)

- **Why:** Many GitHub tools or Docker images are long to run. Put aliases in `~/.bashrc`.
    
- **Examples given in chapter:**
    
    ```
    echo "alias rustscan='docker run --network=host -it --rm --name rustscan rustscan/rustscan:2.1.1'" >> "/home/kali/.bashrc"
    echo "alias gitjacker='/home/kali/tools/gitjacker'" >> ~/.bashrc
    
    ```
    
- **After editing:** either open a new terminal or `source ~/.bashrc`.

## Lab management tips

- **Use intentionally vulnerable Docker containers** for targets — easy to tear down and reset.
- **Take VM snapshots** after setting up your clean Kali environment (do this often, esp. after major config/tool installs).
- **Tear down / rebuild** frequently to keep the lab clean and reproducible.

## Short summary (one-liners)

- **WhatWeb:** Web tech fingerprinting.
- **RustScan:** Super-fast port scanner (use Docker).
- **Nuclei:** Template-based vuln scanner.
- **dirsearch:** Discover web directories/paths.
- **LES2 / LinEnum / unix-privesc-check:** Local enumeration & kernel/privilege escalation help.
- **Gitjacker:** Recover data from exposed `.git`.
- **pwncat:** Post-exploitation interactive shell manager.
- **Aliases & snapshots:** Make tools easy to run and preserve clean lab state.
